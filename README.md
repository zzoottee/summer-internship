# CMS-панель управления контентом (SSR)

Легковесная и производительная система управления контентом (CMS) с серверным рендерингом (SSR), разработанная для динамического администрирования посадочной страницы. Проект изолирует исходную верстку от прямого изменения контента, вынося все управляемые блоки в локальное JSON-хранилище.

## 🚀 Стек технологий

*   **Бэкенд:** Node.js, Express.js (v4+)
*   **Файловый загрузчик:** Multer (для обработки бинарных потоков изображений)
*   **Шаблонизатор:** EJS (Embedded JavaScript templates)
*   **Фронтенд:** Чистый JavaScript (ES6+), HTML5, CSS3

## ⚙️ Архитектурные особенности и возможности

1.  **Server-Side Rendering (SSR):** Страница компилируется полностью на стороне сервера на основе актуального слепка базы данных. Это гарантирует мгновенную скорость загрузки и 100% SEO-оптимизацию сайта поисковыми роботами.
2.  **Single Source of Truth:** Все изменяемые данные централизованно хранятся в единой локальной JSON-базе данных, защищенной от прямого доступа клиентов из интернета.
3.  **Кольцевая (циклическая) сортировка:** Встроенный алгоритм на клиенте позволяет циклически менять порядок элементов списка («вверх-вниз» по замкнутому кругу). Порядок элементов сохраняется на бэкенде автоматически без сбоя индексов.
4.  **Интеллектуальный парсинг контента:** Администратору запрещено вводить сырые HTML-теги. Система автоматически преобразует обычные переносы строк (Enter) в теги `<br>` или `<p>`, а строки, начинающиеся с дефиса (`-`) или звездочки (`*`), парсит в валидные маркированные списки `<ul><li>`. При повторном открытии админки HTML автоматически вычищается для удобного человеческого редактирования.
5.  **Гибридный менеджер медиафайлов:** Система позволяет администратору указывать пути к картинкам как вручную в виде текста, так и загружать новые файлы портретов или логотипов напрямую с компьютера. Модуль `multer` автоматически исключает конфликты имен файлов, сохраняя их в изолированную директорию `/public/uploads/`.

## 🛠️ Инструкция по развертыванию и запуску

Для локального запуска проекта на вашем персональном компьютере выполните следующие пошаговые действия:

### Шаг 1. Проверка системного окружения
Убедитесь, что в вашей операционной системе установлена среда выполнения Node.js. Откройте терминал (Командную строку, PowerShell или Bash) и выполните команды:
```bash
node -v
npm -v
```
*Если команды возвращают ошибку, скачайте и установите Node.js (рекомендуется версия LTS) с официального сайта [nodejs.org](https://nodejs.org).*

### Шаг 2. Скачивание проекта и переход в директорию
Откройте ваш терминал и перейдите в корневую папку, где расположены файлы проекта (`server.js`, `package.json`):
```bash
cd /путь_к_папке_проекта/
```

### Шаг 3. Установка зависимостей
Запустите пакетный менеджер для автоматического скачивания и установки всех необходимых библиотек, зафиксированных в манифесте приложения:
```bash
npm install
```
*После завершения в корне проекта появится папка `node_modules`.*

### Шаг 4. Запуск локального сервера
Запустите исполняемый скрипт бэкенд-сервера:
```bash
npm start
```
При успешном запуске в окне терминала отобразится служебное сообщение:  
`Сервер запущен на http://localhost:3000`

### Шаг 5. Работа в браузере
Не закрывая консоль с работающим сервером, откройте любой современный веб-браузер и перейдите по адресам:

*   **Публичная страница (Сайт):** `http://localhost:3000` — отрендеренная целевая страница, использующая ваши оригинальные скрипты, шрифты, стили и плагин Swiper для анимации слайдов.
*   **Административная панель:** `http://localhost:3000/admin` — интерфейс для мгновенного добавления, удаления, редактирования и изменения порядка блоков. Все изменения вступают в силу сразу после нажатия кнопки «Сохранить» и обычного обновления публичной страницы (F5).
